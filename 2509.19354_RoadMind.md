# RoadMind: Towards a Geospatial AI Expert for Disaster Response

**Paper ID:** 2509.19354  
**Authors:** Ahmed El Fekih Zguir, Ferda Ofli, Muhammad Imran  
**Paper Link:** https://arxiv.org/abs/2509.19354  
**Domain:** NLP · Geospatial AI · Disaster Response · LLM Fine-tuning  



## Overview

**RoadMind** is a geospatially grounded Large Language Model (LLM) framework designed to enhance spatial reasoning for disaster response scenarios. By leveraging structured data from **OpenStreetMap (OSM)**, RoadMind enables LLMs to internalize road network knowledge such as distances, directions, connectivity, and metadata without relying on external tools or internet access.

This approach bridges the gap between traditional geospatial AI systems and language models, enabling **offline-capable, spatially-aware AI assistants** for critical disaster-response tasks.



## Motivation

Effective disaster response requires accurate reasoning over road networks for:

- Evacuation planning  
- Emergency routing  
- Infrastructure damage assessment  

While existing geospatial tools often require expert users and stable connectivity, standalone LLMs lack precise spatial grounding. **RoadMind** addresses this limitation by embedding structured geospatial knowledge directly into LLMs through self-supervised training.



## Core Idea

RoadMind introduces a **modular training framework** that converts structured OSM data into supervision signals for LLMs, enabling them to:

- Interpret geographic coordinates and road networks  
- Estimate distances and directions  
- Reason about spatial relationships without external APIs  

Unlike tool-augmented methods, RoadMind **internalizes geospatial knowledge** via pretraining and instruction fine-tuning, making it suitable for **resource-constrained and offline deployments**.



## Methodology

### 1. Geospatial Data Integration

#### Data Sourcing
- **Primary Source:** OpenStreetMap (OSM)
- **Cities Used:**
  - Los Angeles (large, English, well-represented)
  - Christchurch (medium, English, moderately represented)
  - Metro Manila (large, Tagalog-influenced, underrepresented)
- **Extraction Tool:** `osmnx`

#### Data Structuring
- OSM road *ways* split into segments based on consistent attributes
- Extracted attributes per segment:
  - Geometry (latitude, longitude tuples)
  - Road name
  - Road type (`highway` tag)
  - Speed limit
  - Lane count
  - Segment length (Haversine distance)
- Coordinate system: **WGS84 (EPSG:4326)**



### 2. Geospatial Representations

#### Core Representations
- **Point-to-Segment (P2S)**
- **Segment-to-Info (S2I)**
- **Road-to-Info (R2I)**

#### Spatial Reasoning Representations
- Road-to-connected-roads
- Point-pair-to-distance
- Point-pair-to-direction (8 cardinal directions)
- Point-to-directional-roads



### 3. Data Augmentation

- **Density-aware spatial sampling** using grid-based road density
- **Directional nearest-road retrieval** for spatial diversity



### 4. Training Data Formats

Each representation is converted into:

- **Pretraining format:** Wikipedia-style factual text
- **Instruction fine-tuning format:**  
  - ≥5 diverse prompt templates per input-output pair



### 5. Model Adaptation (QLoRA)

- **Base Models:**
  - LLaMA 3.1 8B
  - Mistral 7B v0.3
  - Qwen 3 8B
- **Technique:** QLoRA (4-bit quantization + LoRA adapters)
- **Two-stage training:**
  1. Continual pretraining
  2. Instruction fine-tuning



## Evaluation

### Tasks
- Road metadata prediction
- Distance and direction estimation
- Nearest and directional road retrieval

### Baselines
- Open-source LLMs (prompting only)
- GPT-4o Mini (prompt-based)
- Advanced prompting strategies:
  - Chain-of-Thought
  - Few-shot learning
  - Query-Specific Few-shot (QSF)


## Results

- **Prompting alone is insufficient** for geospatial reasoning
- RoadMind shows significant improvements:
  - Road retrieval: **150–430%** improvement
  - Distance estimation: **MAPE < 0.032**
  - Direction prediction: **F1 up to 0.95**
  - Road name accuracy tripled (Los Angeles)
- Largest performance gains observed even in data-rich cities


## Key Takeaways

- Structured geospatial data significantly enhances LLM spatial reasoning
- Self-supervised OSM-based pretraining enables offline geospatial intelligence
- QLoRA makes large-model adaptation practical
- RoadMind demonstrates strong potential for disaster response applications
